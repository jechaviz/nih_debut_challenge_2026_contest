# Clinician Validation Protocol

Purpose: add credible, low-burden validation evidence before final submission.

## Participants

- One faculty sponsor or clinical advisor.
- One clinician, nurse, community health worker or kidney-care stakeholder.
- Undergraduate evaluation lead.

## Session length

30-45 minutes.

## Materials

- RenalCue web demo.
- Eight synthetic case receipts.
- One-page fact sheet.
- Short note explaining that no PHI should be entered.

## Procedure

1. Show the current community CKD screening workflow.
2. Ask the participant where delays or missed handoffs occur.
3. Run three synthetic cases: low risk, nephrology referral and urgent review.
4. Ask whether the receipt is clinically understandable.
5. Ask what wording would make the tool safer or clearer.
6. Record only non-private feedback and action items.

## Success criteria

- Participant can explain the risk receipt without developer help.
- Participant agrees the tool could reduce triage ambiguity in a supervised
  screening workflow.
- Participant identifies no unsafe autonomous-diagnosis claim.
- At least one concrete UI, wording or workflow improvement is captured.

## Evidence to include

- Date, role and organization type of reviewer.
- Non-identifying summary quote.
- Before/after workflow pain point.
- Changes made after feedback.
